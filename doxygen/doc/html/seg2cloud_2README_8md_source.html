1 # seg2cloud
2 Module to capture pointclouds from segmented regions in 2D or 2.5D (depth) images
3 
4 ## Description
5 This module combines disparity and segmentation information in order to retrieve 3D clouds of desired 2D regions. These regions can be chosen from a user defined polygon, from a segmentation algorithm, or from 2D (in color) or 3D (in depth) flood filling. When using segmentation modules (2D or 3D), seeding can be performed automatically.

14 ## Reconstruction Options
15 This modules interfaces between segmentation modules and SFM in order to provide the 3D reconstruction of an object delimited by a contour in the image. In order to obtain this contour, 4 modes are provided:
16 
17 - **seg:** 2D segmentation (requires external segmentation module). When this option is called, the contour is provided by an external segmentation module.
18 - **flood:** 2D color flood. The object's contour is obtained by performing a 2D color flood, based on the thresholding parameter *color_distance*.
19 - **flood3D** 3D depth flood. The contour is determined by a flood operation performed on the depth image, controlled by the thresholding parameter *space_distance*.
20 - **polygon** User selected polygon contour. The contour is provided by the user as a contour by selecting the vertices on a connected yarpview window.

23 ## Seed feeding options
24 In order to select which of the possible regions of the image is segmented, a seed usually needs to be provided. In this module, the seed can be provided in several different ways:
25 - In the three first options above (`seg`, `flood` and `flood3d`) the initial seed to perform segmentation can be provided in 3 different ways:
26 - If parameter `seedAuto` is set to *true*, the seed will be obtained automatically.
27  - In segmentations options `seg` and `flood`, seed has to be provided by a tracker whose ooutput (u v coordinates) is connected to the `/seg2cloud/seed:i` port.
28  - If the segmentation option is `flood3d`, the seed is computed automatically as the center of the closest blob (as in the disbBlobber module).
29 - If parameter `seedAuto` is set to *false*, the seed can be coveyed in 2 further ways:
30  - By clicking on a connected yarpview window whose output is connected to the `/seg2cloud/seed:i`port.
31  - As *explicit u v coordinates* given after the segmentation command. For example <tt> seg 102 175 </tt>.
32 - In the case where the segmentation option is set to `polygon`, no seed is needed, as the region is fully provided by the user.
